# Бэкап стоковой прошивки камеры BK7252N

> Это **первый и обязательный** шаг перед любым реверсом/перепрошивкой.
> Дамп нужен по двум причинам:
> 1. **Страховка** — если что-то пойдёт не так, мы зальём его обратно и камера оживёт.
> 2. **Топливо для реверса** — в стоковой прошивке уже есть рабочий код камеры под
>    именно твой BK7252N. Из неё я вытащу реальные адреса регистров DVP/JPEG/I2C,
>    последовательность инициализации сенсора и раскладку флеша. Без этого порт
>    делать вслепую.

⚠️ **Пока ничего НЕ пишем в камеру. Только читаем.**

---

## 1. Что понадобится

- **USB-UART переходник на 3.3 В** (CP2102 / CH340 / FT232 — любой). Важно: уровни 3.3 В, не 5 В.
- Тонкие провода / паяльник с тонким жалом.
- ПК (у тебя Windows).
- Софт (любой из, по убыванию удобства):
  - **BKwriter / `beken_write_V1.45`** — GUI, лежит в репозитории:
    `beken7252-opencam\bdk_rtt\tools\beken_write_V1.45` (есть кнопка чтения флеша).
  - **`bk7231tools`** (Python, community) — самый надёжный для чтения дампа:
    `pip install bk7231tools` (поддерживает чтение всего флеша одной командой).
  - **`uartprogram`** из репозитория: `beken7252-opencam\hid_download_py\uartprogram`.

---

## 2. Найти UART на плате

Beken-загрузчик (bootrom) общается по UART1. На плате A9 ищи:
- площадки/тест-пойнты рядом с чипом, иногда подписаны **RXD / TXD / GND / 3V3** (или R/T/G);
- если подписей нет — TX чипа обычно «болтается» в районе 3.3 В в покое, GND звонится на минус;
- питание камеры на время прошивки удобно подать с **3.3 В переходника** (или от родного аккумулятора/USB — но тогда GND всё равно объединить с переходником).

**Распайка (крест-накрест):**

| USB-UART | Камера (BK7252N) |
|----------|------------------|
| TX       | RXD              |
| RX       | TXD              |
| GND      | GND              |
| 3V3      | 3V3 (если питаешь от переходника) |

> Если найдёшь только две точки UART — сними фото платы крупным планом и пришли,
> помогу опознать TX/RX/GND и сам чип (подтвердим маркировку `BK7252N…` и объём флеша).

## 3. Считать дамп

Beken-bootrom слушает UART **только первые миллисекунды после включения**, поэтому
порядок такой: **сначала запускаем чтение в программе, потом подаём питание на камеру**
(или жмём reset). Программа «ловит» загрузчик в момент старта.

### Вариант A — bk7231tools (рекомендую)
```bat
:: подставь свой COM-порт (см. Диспетчер устройств)
:: читаем весь флеш (обычно 2 МБ = 0x200000) в файл
python -m bk7231tools read_flash -d COM5 -s 0 -c 0x200000 stock_bk7252n_backup.bin
```
Если объём флеша другой — инструмент обычно сам сообщит размер; считай ровно весь объём.

### Вариант B — BKwriter GUI (`beken_write_V1.45`)
1. Запусти `BKwriter.exe`.
2. Выбери COM-порт и скорость (начни со `115200`, если ловится плохо — `921600`).
3. Режим **Read / Read Flash**, диапазон `0x0` … конец флеша (2 МБ).
4. Нажми чтение → **сразу подай питание/reset на камеру**.
5. Сохрани результат в `.bin`.

### Вариант C — uartprogram (из репо)
Утилита заточена в основном под запись, но в комплекте `hid_download_py` есть и чтение —
если выберешь этот путь, напиши, дам точную команду под текущую версию из репозитория.

---

## 4. Проверить и сохранить дамп

```bat
:: размер должен совпасть с объёмом флеша (например ровно 2097152 байт для 2 МБ)
dir stock_bk7252n_backup.bin
```
- Сделай **две копии** в разные места (и желательно в облако/на другой диск).
- Признаки хорошего дампа: размер ровно равен флешу; внутри видны строки
  (имя SDK, версия, `rtos`, имена разделов). Если файл весь из `0xFF` или `0x00` —
  чтение не удалось, повтори с другой скоростью / лучше пропаяв контакты.
