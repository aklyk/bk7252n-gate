# Чем прошивать BK7252N (железо + софт)

> ⚠️ **Сначала backup!** До любой записи сними полный дамп стоковой прошивки
> (см. `../how_to_backup_bk7252n.md`). Это страховка от кирпича. Записывать — только после бэкапа.

## Железо (что купить/найти)

| Что | Зачем | Примечание |
|---|---|---|
| **USB-UART переходник 3.3 В** | основной способ прошивки BK7252N (через bootrom по UART1) | CP2102 / CH340 / FT232. Уровни **3.3 В**, не 5 В! |
| Тонкие провода + паяльник | припаять TX/RX/GND к плате камеры | площадки UART рядом с чипом |
| (опц.) лаб. БП или питание 3.3 В | питать камеру при прошивке | можно от переходника 3V3 |
| (опц.) USB-логический анализатор | если будем сверять I2C-инициализацию сенсора | помогает на этапе отладки картинки |

Распайка (крест-накрест): переходник **TX→RXD** камеры, **RX→TXD**, **GND→GND**, при питании от переходника **3V3→3V3**.

## Софт для прошивки (любой из)

| Инструмент | Платформа | Чтение (backup) | Запись |
|---|---|---|---|
| **bk7231tools** (`pip install bk7231tools`) | Win/Lin/Mac, Python | ✅ надёжно | ✅ |
| **BKwriter / `beken_write_V1.45`** (GUI) | Windows | ✅ | ✅ |
| `hid_download_py` / `uartprogram` (из репо opencam) | Win/Lin | ✅/частично | ✅ |
| `beken_program_V2.5.4` (GUI, из `bdk_rtt/tools/`) | Windows | ✅ | ✅ |

Все они шьют по UART, ловя bootrom в момент включения: **запускаешь запись → подаёшь питание/reset на камеру**.
Скорость начинать со `115200`, при плохом коннекте поднять до `921600`.

## Что именно будем шить

- Образ из сборки: `bdk_rtt/out/*.bin` (см. `BUILD.md`), упакованный в формат загрузчика Beken.
- Адрес записи: как в стоке — app по логическому `0x10000` (точные смещения возьмём из таблицы
  разделов FAL твоей камеры; в стоке: bootloader/app/download/filesystem/EasyFlash).
- **Bootloader не трогаем** — шьём только раздел app (и при необходимости filesystem/EasyFlash).
  Так загрузчик останется цел, и при неудаче всегда можно вернуть бэкап.

## Порядок действий (когда будет бинарь)

1. Бэкап стока (полные 2 МБ) → 2 копии.
2. Записать новый образ в раздел app.
3. Открыть UART-терминал (115200) → включить камеру → читать лог загрузки.
4. По логу/картинке правим драйвер (`/*VERIFY*/` биты) → пересборка → перешивка. Цикл.
5. При проблеме — заливаем бэкап обратно, камера жива.

## Восстановление (анти-кирпич)

Если камера не грузится — заливаем сохранённый `stock_..._backup.bin` целиком тем же инструментом.
Bootrom BK7252N всегда слушает UART при старте, поэтому «насмерть» окирпичить через UART крайне трудно.
