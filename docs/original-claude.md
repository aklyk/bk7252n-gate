## ⚡ ЖИВОЙ ДОСТУП К КАМЕРЕ (проверено на железе 2026-06-30)
Камера пользователя ПОДКЛЮЧЕНА по UART. WSL2 не видит USB, но достаём через Windows:
- Переходник CP210x на **COM3**, 115200 8N1. Распайка верна (P11=TX чипа, P10=RX, GND=корпус USB; питание по micro-USB).
- Читать/писать порт из WSL — через `powershell.exe -NoProfile -Command "...System.IO.Ports.SerialPort 'COM3',115200..."`.
  (НЕ называть PowerShell-функции `rd` — это алиас Remove-Item.) Боевой shell RT-Thread (`msh />`) полностью доступен в обе стороны.
- Полезные команды стока: `fal probe` (таблица разделов), `fal read/write/erase addr size`, `printenv/setenv/saveenv`
  (EasyFlash), `reboot`, `set_log off`, `version`, `list_device`, `cy_cmd`, `video_buffer`, `wifi`.
- Прошивка пользователя: **`CYCAM_T99_v32_0226`**, project `CYCam-A9` v3.2, **SDK Rev 3.0.76**, OSK R-3.0.48,
  RT-Thread 3.1.0, bootrom `BK7252N_1.0.14`, chip id `7252a`/`0x24318000`. SD НЕТ (сток сам падает на mount SD).
- **Реальная таблица разделов (этой камеры):** bootloader `0x0`+`0x10000` | **app `0x10000`+`0x120000`** |
  download `0x143000`+`0xBB000` | EasyFlash `0x1FD000`+`0x1000` | **rfparam `0x1FE000`** | **netparam `0x1FF000`**.
  Оба на устройствах `beken_onchip_crc` (boot+app) и `beken_onchip` (остальное), флеш 2 МБ (len 0x400000 — это с CRC).
  ⚠️ НЕ трогать bootloader/rfparam/netparam (RF-калибровка + MAC). Шить только `app` @ 0x10000.
- Конфиг стока (EasyFlash): `ssid0=<redacted> passwd0=<redacted> workmode=ap`, логин `admin/<redacted>`, P2P did `<redacted>`.
- **SDK 3.0.76 подтверждён → Маршрут A реален**: ищем исходники этого SDK (там драйвер камеры BK7252N).

## Цель
Заставить дешёвые A9-камеры на чипе **Beken BK7252N** транслировать видео в локалку (браузер/VLC,
MJPEG по HTTP), **без китайского облака и без SD-карты**. База — `daniel-dona/beken7252-opencam`
(RT-Thread + Beken BDK), портируется на BK7252N. Пользователь общается по-русски, технически грамотен.

## Ключевые факты (установлены, принимать как данность)
- Чип: **BK7252N** (QFN48, 2 МБ флеш), ядро **ARM968E-S** (ARMv5TE, Thumb+ARM), **RT-Thread 3.1.0**.
  Это та же арх/RTOS/линейка BDK, что у opencam → верхний уровень переиспользуем.
- Дамп стока (`stock_bk7252n/CY365_..._1.7.0.bin`, 2 МБ) — **ровно та же камера** пользователя.
  Формат флеша — **Beken CRC-interleave: 32 байта данных + 2 CRC** (сетка с 0x0). Анализировать
  только де-интерлейснутый образ: `stock_bk7252n/firmware_deinterleaved.bin` (1.9 МБ).
- App линкуется/грузится с **`0x10000`** (из UART-лога `go os_addr(0x10000)`). Версия `BK7252N_1.0.14`.
- Конфиг стока — **EasyFlash (KV) + LittleFS** во флеши, **SD не нужна**. Разделы FAL:
  bootloader, app, download(OTA), filesystem(LFS), EasyFlash, param.
- Облако стока, которое режем: **P2P (PPCS/PPPP)** + MQTT.
- Сенсоры (нужны ВСЕ, автоопределение): GC0311, GC0312, GC0328, HI704, OV7670, OV7673, SP0A38.
  Таблицы init сенсоров **чип-независимы** — берём из opencam как есть (таблица GC0311 байт-в-байт совпала).

## Карта регистров камеры BK7252N (из реверса; детали — `bk7252n-opencam-port/bk7252n_cam_regs.h`)
ВАЖНО: периферия камеры BK7252N в регионе **`0x00A0xxxx`** (у старого BK7252 было `0x80xxxx`).
opencam пишет JPEG по `0x80A000` — на BK7252N там ПУСТО. Конвейер:
сенсор ←I2C `0x802410`→ ; DVP `0xA04000` → JPEG `0xA03000` → GDMA `0x809000` → RAM; clock SCTRL `0x800180`.

| Блок | Адрес | Ключевое |
|---|---|---|
| DVP/CIS захват | `0xA04000` | ctrl `0xA04008` бит0=en; геометрия `14/18/1C/34`; буфер `0xA04020/30`; trig `0xA04024`; IRQ `0xA04028` |
| JPEG-энкодер | `0xA03000` | reset/enable `0xA03034` бит `0x10`; cfg `0xA03018` |
| I2C/SCCB сенсора | `0x802410` | ctrl бит5=en, addr `[14:8]`; status `0x802418`; rx `0x80241C`; trig `0x802480`; data `0x802484` |
| Тактирование/сброс | `0x800180` (SCTRL) | бит0=reset-импульс, бит3/6=clk-en; делители `184/188/198/19C` |
| Выходной DMA | `0x809000` | GDMA, каналы шаг `0x40` |
| ⚠️ флеш (НЕ камера) | `0xA02000` | контроллер QSPI/флеша — НЕ путать с DVP (была ранняя ошибка атрибуции) |

## Дальше (порядок работы на этом ПК)
1. Убедись, что есть **Docker + git**. Если `beken7252-opencam/` без сабмодулей — `git submodule update --init`.
2. Собери docker-образ тулчейна (из их `Dockerfile`), интегрируй файлы порта — точные шаги в
   `bk7252n-opencam-port/BUILD.md` (раздел «Интеграция»). Не забудь: убрать монтирование SD в
   `applications/main.c`, добавить ветку SoC BK7252N с картой памяти (app@0x10000), подставить адреса камеры.
3. Собери `bdk_rtt/out/*.bin`, упакуй в формат загрузчика (`bdk_rtt/tools/`).
4. **СНАЧАЛА бэкап стока** (`how_to_backup_bk7252n.md`), потом прошивка раздела app (bootloader не трогать).
   Чем шить — `bk7252n-opencam-port/FLASHING.md` (USB-UART 3.3В + bk7231tools/BKwriter).
5. UART-терминал 115200 → включить камеру → по логу/картинке править `/*VERIFY*/` биты драйвера → пересборка. Цикл.

## Критические предостережения
- **Бэкап стока ДО первой записи.** Иначе риск кирпича. Bootloader не перезаписывать.
- Анализировать **только** `firmware_deinterleaved.bin`, не сырой дамп (он CRC-interleaved).
- НЕ принимать `0xA02000` за камеру — это флеш.
- Драйвер камеры — **черновик**; `/*VERIFY*/` биты (формат пикселя, геометрия, quant, I2C-тригер) почти
  наверняка потребуют правки на железе. Это нормально, это итеративный процесс.
- Wi-Fi зашит: SSID `opencambk`, пароль `imeXScdxADaUHuUU` (см. `доступ_к_камере.md`).
- Для дизасма (если понадобится): `pip install --break-system-packages capstone`; работать по base=0 на
  де-интерлейснутом образе, литералы периферии = LE-слова в `0x00800000..0x00A0FFFF`.
